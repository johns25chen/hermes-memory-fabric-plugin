# 文明之核：十五层记忆总框架

状态：v2.10-pre 框架纠偏文档
范围：文明之核、十五层记忆、亚空间记忆系统、Hermes、Codex、OpenClaw、GitHub
性质：最高记忆坐标系校准；不新增工具、不写入 durable memory、不改变 release tag

## 1. 总定位

文明之核是总系统、总宪法、总操作内核。它定义长期记忆、证据、审计、执行、
授权、版本封印和多智能体协作之间的最高治理秩序。

十五层记忆是文明之核的最高记忆等级体系。它不是工程模块清单，也不能被
Hermes 插件层、Codex 工作流层、OpenClaw 执行层或 GitHub 发布层替代。

亚空间记忆系统是十五层记忆的工程实现载体。它把最高记忆等级体系落实为可
验证、可审计、可分区、可评审、可回滚的受治理记忆结构。

Hermes、Codex、OpenClaw、Terminal 和 GitHub 是组件，不是核心。它们分别承担
记忆治理、代码与文档实现、执行面观察或操作、终端验证、版本链封印等职责，
但任何组件都不得把自身流程提升为文明之核本体。

## 3. 三段结构

1-5 是记忆生成层。该段负责从原始意图、单点事实、证据链、结构图到项目演化
流的形成，重点是来源、结构和历史连续性。

6-10 是知识治理与协同层。该段负责知识沉淀、项目和角色分区、审计治理、联
邦共享、大规模召回以及跨系统协同，重点是受治理的记忆质量和边界协作。

11-15 是元治理与自演化层。该段负责调度编排、自我规则、长期人格和偏好连续
性、跨时间跨生态演化，以及来源方法论自进化，重点是高阶治理能力，而不是当
前工程功能的简单堆叠。

## 4. 当前工程位置

v2.9.0 工程上实现 closed-loop evidence validation。在十五层记忆体系中，它
对应星界记忆入口。

当前工程已具备星辰、星域、星穹基础。星海正在形成，表现为多来源、多组件、
多证据面的召回、审计和闭环验证结构开始汇合。

星枢、星律、星魂、星宙、星源尚未完成。当前存在边界规则和审计约束，但不能
宣称已经形成成熟的中心调度、自执行记忆法律、长期人格连续性、跨时间跨生态
演化记忆，或最高层来源方法论自进化。

## 5. 推进规则

未来每一个版本都必须明确说明：

- 文明之核总目标。
- 本版本对应十五层记忆中的哪一层，且必须指定一个主层级。
- 本版本工程目标。
- 本版本允许动作。
- 本版本禁止越界。
- 是否需要 Codex CLI。
- 是否需要 OpenClaw。
- 是否可以打 tag。

任何版本不得用工程层、工具层、PR 流程或发布流程替代十五层记忆坐标。任何版
本也不得因为完成了局部验证，就宣称文明之核或十五层记忆体系已经完成。
