## 2. One-Sentence Product Narrative / 一句话产品叙事

Civilization Core is a governance kernel and memory constitution for AI memory systems, separating source, evidence, candidate, review, approval, adoption, and execution so long-term memory can remain traceable, scoped, and under human sovereignty.

It is not autonomous authority, a self-authorizing agent, a finished commercial product, an active Layer 15 runtime, an active Star-Source runtime, a legal subject, personhood, life, awakening, or a religious entity.

## 3. Problem Statement / 问题定义

AI memory systems fail when useful output is mistaken for governed memory. The product problem is not only how to store or retrieve context; it is how to decide what may become trusted, durable, scoped, reviewable memory.

Key non-equivalences:

| Capability | Product risk if overread |
| --- | --- |
| AI can generate information. | Generation is not trusted memory. |
| AI can retrieve information. | Retrieval is not memory adoption. |
| A database can save data. | Storage is not governance. |
| RAG can return content. | Return is not authorization. |
| An Agent can call tools. | Tool use is not lawful execution. |
| Tests can pass. | Test success is not implementation permission. |
| Audits can find issues. | Audit finding is not repair authorization. |

Without governance boundaries, long-term memory can create contamination, drift, overreach, and untraceable decisions. The system may remember too much, remember the wrong source, collapse review into approval, or let a tool surface behave as if it were a human authority.

## 4. Core Value Proposition / 核心价值主张

Civilization Core's product-facing value is governed memory: memory that preserves evidence provenance, keeps candidate-only material separate, separates review from approval, supports scoped approval, and protects human sovereignty.

Core value claims:

- evidence provenance makes memory explainable;
- candidate-only handling prevents premature persistence;
- review separation prevents inspection from becoming authority;
- scoped approval prevents broad future overreach;
- human sovereignty keeps final control with humans;
- non-automatic persistence blocks hidden durable writes;
- non-automatic execution blocks hidden action;
- explainable decision history supports audit and handoff;
- bounded handoff transfers context without transferring authority;
- anti-drift discipline protects the sealed boundary.

Memory quality alone is insufficient. Governance quality is a separate requirement. More memory is not automatically better memory. Faster execution is not automatically safer execution.

## 9. Example User Journeys / 示例用户旅程

### Journey A: Personal Knowledge Candidate

Flow: source -> evidence -> candidate -> human review -> scoped decision.

A person captures a useful note from a source. Civilization Core frames it as evidence-backed candidate memory. A human reviews source quality, role scope, and deletion or revision needs before making a scoped decision. There is no automatic adoption.

### Journey B: Team Decision Handoff

Flow: meeting record -> evidence package -> review -> handoff context.

A team records a decision and packages the evidence for a later operator. The handoff context explains what was discussed, what is still uncertain, and what must be reviewed. Handoff is not execution authorization.

### Journey C: Enterprise Knowledge Review

Flow: source registration -> provenance inspection -> policy review -> scoped approval request.

An enterprise governance team registers a source, inspects provenance, checks policy boundaries, and prepares a scoped approval request. The approval request is not approval.

### Journey D: Agent Memory Proposal

Flow: agent observation -> bounded evidence -> memory proposal -> human review.

An Agent observes a repeated fact pattern and prepares bounded evidence. It may propose memory for review. It may not automatically persist, approve, authorize, or execute.

## 11. Differentiation Narrative / 差异化叙事

Civilization Core does not replace databases, vector stores, RAG systems, knowledge graphs, agent memory, workflow engines, orchestration frameworks, audit logs, or access-control systems. It provides governance boundaries, an explanation framework, and a human-control layer around memory-bearing systems.

| Category | What it may do | Civilization Core difference |
| --- | --- | --- |
| Database | Store records. | Storage does not decide memory legitimacy. |
| Vector store | Retrieve similar content. | Similarity is not adoption or approval. |
| RAG system | Return grounded answers. | Returned content is not authorized memory. |
| Knowledge graph | Model entities and relationships. | Graph structure is not governance or mutation permission. |
| Agent memory | Carry context across tasks. | Agent context is not self-authorizing memory. |
| Workflow engine | Move tasks through steps. | Workflow progress is not human approval. |
| Orchestration framework | Route tools and agents. | Routing is not lawful execution authority. |
| Audit log | Record events. | Audit evidence is not remediation permission. |
| Access-control system | Control who can access what. | Access is not adoption, approval, or execution. |

The differentiation is governance, not replacement.
