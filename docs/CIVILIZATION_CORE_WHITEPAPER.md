# Civilization Core Whitepaper / 文明之核白皮书

## 1. Abstract

Civilization Core is a governed memory and coordination architecture. It defines how memory, evidence, review, approval, execution boundaries, audit records, and long-term continuity should be separated and governed in systems that carry durable context.

Subspace Memory System is the engineering carrier for that architecture. It gives the governance model a concrete project surface without reducing the model to one plugin, one workflow, one script, or one repository automation path.

The Civilization Core core governance line from v1 through v6 closed at `v6.16.0`. `v6.16.0` is the Civilization Core Stable Kernel. That kernel is sealed as governance metadata and boundary definition, not as an operating executor.

The marker after `v6.16.0` is `v6_series_terminal_boundary`. This marker is terminal metadata only. It is not `v6.17`, not a release stage, and not a continuation of the v6 line.

## 2. Problem Statement

AI systems can retrieve, summarize, reason over context, and act through tools. Those abilities become unsafe or incoherent when memory, evidence, approval, execution, audit, and long-term continuity are treated as the same surface.

Civilization Core addresses that problem by making governance distinctions explicit:

- memory recall must not equal memory write;
- audit must not equal authorization;
- approval request must not equal approval;
- execution adapter declaration must not equal execution;
- methodology reasoning must not equal self-authorization;
- stable kernel metadata must not equal active runtime.

Without these distinctions, a system can confuse seeing evidence with accepting evidence, requesting authority with receiving authority, declaring an adapter with dispatching work, or describing a kernel with running a kernel.

## 3. Core Thesis

Civilization Core is not merely a memory plugin, not merely a RAG tool, not merely an agent bus, and not merely a workflow.

It is a governance constitution for memory-bearing systems.

It defines what memory can recall, what evidence can support, what proposals can request, what reviews can approve, what execution boundaries must forbid, and what humans must retain authority over.

The architecture is therefore concerned less with adding capability than with deciding when capability is not authorized, when evidence remains insufficient, when review remains non-executing, and when a boundary must stay closed.

## 4. System Identity

Civilization Core is the top-level governance architecture.

Subspace Memory System is the engineering carrier.

Hermes, Codex, OpenClaw, Terminal, and GitHub are components or operators. They may observe, implement, validate, package, publish, or coordinate parts of the system, but they are not the core itself.

The project must not be reduced to a Hermes plugin, Codex workflow, OpenClaw script, Terminal transcript, or GitHub automation. Those surfaces are operational context around the architecture. They do not replace the architecture.
