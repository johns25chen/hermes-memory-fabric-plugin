## 3. Questions and Answers

### 1. What is Civilization Core?

Civilization Core is a governance kernel for AI memory systems. It defines memory, evidence, proposal, review, approval, execution, continuity, and non-overreach boundaries.

### 2. What is Subspace Memory System?

Subspace Memory System is the engineering carrier for the Civilization Core architecture. It gives the governance kernel a project surface without becoming a separate authority source.

### 3. What is Hermes Memory Fabric?

Hermes Memory Fabric is the current repository and implementation carrier for governed memory-fabric work. It is a carrier, not the whole Civilization Core identity.

### 4. Is this just a Hermes plugin?

No. The repository is named as a Hermes Memory Fabric plugin, but Civilization Core is broader: it is the governance architecture and boundary system for memory-bearing AI systems.

### 5. What is `v6.16.0`?

`v6.16.0` is the sealed Civilization Core Stable Kernel. Local docs and git history use it as the final closed v6 governance identity.

### 6. Is `v6.16.0` still open for changes?

No. It remains sealed. Post-terminal docs explain, package, audit, navigate, or discuss future possibilities without changing `v6.16.0` semantics.

### 7. Why is there no `v6.17`?

The v6 line closed at `v6.16.0`. The terminal marker after it is terminal metadata only, not a successor version or continuation target.

### 8. Are T1-T10 releases?

No. T1-T10 are post-terminal documentation surfaces after `v6.16.0`. They are not package releases and do not reopen v6.

### 9. Why are T1-T10 not tagged?

Because they are documentation after the sealed kernel, not new release identities. Tagging them would risk implying a successor release.

### 10. What is the Document Index?

The Document Index is the navigation entry point. It tells readers where to start, what each document is for, and which future-facing docs must not be overread.

### 11. What is the One-Page Overview?

The One-Page Overview is a short explanation layer added for external readers. It makes the project easier to explain without making it more executable.

### 12. What is the Comprehensive Audit Report?

The Comprehensive Audit Report is an audit-only review of the post-terminal document set. It identifies strengths, risks, reader-path gaps, and safe documentation-only next steps.

### 13. Does T9 start v7 implementation?

No. T9 is v7 pre-design decision material only. It does not create a v7 implementation branch or authorize v7 implementation.

### 14. Does T10 authorize product implementation?

No. T10 is a productization roadmap only. It discusses future product narrative and direction without approving product implementation.

### 15. Can we build an MVP from T10?

No. T10 does not authorize an MVP, product deployment, UI implementation, operation surface, durable writer, or execution behavior.

### 16. Is Operator Console approved for implementation?

No. Operator Console is roadmap / candidate material only. It is not an approved UI implementation, execution surface, automatic approval surface, memory writer, or product deployment.

### 17. What are `llm_wiki`, M-Flow, and GBrain in this project?

They are external methodology candidates. `llm_wiki` is a knowledge compiler methodology candidate, M-Flow is an associative recall methodology candidate, and GBrain is a memory operations methodology candidate.

### 18. Are external candidates dependencies?

No. External candidates are not dependencies, adapters, runtime behavior, identity transfer, or v6 completion conditions.

### 19. What does "methodology candidate" mean?

It means the project may study a useful method, vocabulary, structure, or review pattern later. It does not mean importing code, activating capability, or approving implementation.

### 20. What does Layer 15 / Star-Source Memory mean here?

It means a source reasoning and methodology boundary in the sealed governance map. It is metadata and interpretive structure, not a live runtime layer.

### 21. Is there active Layer 15 runtime?

No. Layer 15 is sealed governance metadata and methodology boundary only.

### 22. Is there active Star-Source Memory runtime?

No. Star-Source Memory wording describes sealed governance semantics, not active runtime behavior.

### 23. Does the project claim personhood, life, awakening, legal status, or religious status?

No. The project explicitly does not claim personhood, life, awakening, legal subject status, religious status, autonomous authority, or self-authorization.

Read the Document Index, Release Book, Handoff Package, Boundary Constitution, and Comprehensive Audit Report first. For v7 discussion, also read T9. For product narrative, also read T10.

### 30. What is the short external explanation?
