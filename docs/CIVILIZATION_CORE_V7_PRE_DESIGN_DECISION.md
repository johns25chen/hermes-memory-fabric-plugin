## 8. Knowledge Compilation Candidate

The Knowledge Compilation Candidate is inspired by `llm_wiki` methodology.

Future design may study:

- raw sources;
- compiled knowledge candidates;
- purpose, schema, index, and log discipline;
- review queue;
- lint health;
- graph insight.

Generated knowledge remains candidate.

Compiled knowledge is not approved memory.

Graph insight is not Memory Graph mutation.

There is no automatic ingest.

There is no dependency adoption.

There is no runtime activation.

The borrowable lesson is knowledge compilation discipline: preserve raw inputs, compile derived candidates, expose purpose and schema, maintain logs, queue review, lint the corpus, and surface graph insight without treating generated knowledge as approved memory.

## 9. Associative Recall Candidate

The Associative Recall Candidate is inspired by M-Flow methodology.

Future design may study:

- Episode;
- Facet;
- FacetPoint;
- Entity;
- semantic edge;
- path-cost recall;
- associative evidence path.

Recall candidate remains candidate.

Associative recall is not durable memory adoption.

Semantic edge is not Memory Graph mutation.

Path-cost evidence is not authorization.

There is no M-Flow runtime behavior.

There is no dependency adoption.

There is no adapter.

There is no vector or graph retrieval activation.

The borrowable lesson is associative recall discipline: evidence paths may help a human reviewer find candidates, but the path itself does not write memory, mutate a graph, authorize action, or execute anything.

## 10. Memory Operations Candidate

The Memory Operations Candidate is inspired by GBrain methodology.

Future design may study:

- answer + citations + gaps;
- Brain / Source boundary;
- operation contract;
- localOnly / remote / scope trust boundary;
- doctor / health report;
- eval / replay / benchmark.

Operation contract is not dispatch.

Doctor report is not remediation.

Eval pass is not authorization.

There is no MCP/API operation activation.

There is no dependency adoption.

There is no adapter.

There is no remote operation surface.

The borrowable lesson is memory operations discipline: answers should cite evidence, expose gaps, separate brain from source, define trust boundaries, and remain reviewable without creating dispatch, admin, write, remote, authorization, or execution surfaces.
